Scopes in JavaScript

Global Scope Vs Local Scope

Global Scope

Note! Ager kisi Variable ko "Let" yan "Const" sy define Karyn gy tu wo "Script Scope" main store ho ga. Or ager "Var" sy define Karyn gy tu wo "Window Object" main store ho ga.

Note! Ager hum kisi Variable ko "Function" ky bahir Define karyn tu usy Function ky ander b "Console" Karwa Sakty hain. But ager "Variable" ko Function ky ander Define Karyn tu usy Function ky ander hi "Console" Karwa Sakty hain. Function ky bahir "Console" nai Karwa Sakty.

Question: Global Scope ky ander "Script" kab aye ga?
Answer: Global Scope ky ander "Script" usi time aye ga jab hum "Let" yan "Const" sy "Variable" define Karyn gy.

Note! Global Scope ky ander 2 Categories hoti hain. in dono ky ander jo b "Variable" hoon gy unhyn hum apny "Program" main kahi b "Use" kar Sakty hain.
  1. Script Scope
  2. Window Object
Question: In dono main kia difference hota hai?
Answer: In main difference yeh hai ky jo b "Variable" "Let" yan "Const" sy define Karty hain wo "Script Scope" main store hoty hain. Or jo "Var" sy define Karyn gy wo "Window Object" main store ho ga.

Note! "Let" or "Const" sy jo "Variable" define Karyn gy or jab tak un main "Undefined" hota hai tu wo "Temporal Dead Zone (TDZ)" main hoty hain.

Note! "Window Object" main jo b ho ga usy hum "Window." sy access kar Sakty hain. "Otherwise" access nai kar Sakty.

Local Scope

Note! Jab hum kisi "Function" ko "Call" karty hain tu us ka alg sy ik "Scope" Create hota hai. Or us ka Name "Local Scope" hota hai. Or is ky Start main "This Object" Create ho ga Phir baqi "Variables" wagera. Or jesy hi Function "End" ho ga "Local Scope" b "Delete" ho jye ga. Or us ky ander sy sab "Variables" b "Delete" ho jaty hain. Isi lye Function ky ander Variables ko hum Function ky bahir access hai kar Sakty. Kio ky wo sab Delete ho jaty hain.

Note! Ager hum Function ky ander koi "Variable" define karyn Same name ky sath jo hum ny pehly b define kia ho. Tu ager hum us "Variable" ko access karyn gy. Tu access wahi ho ga jo "Function" ky ander ho ga. Kio ky yeh apny Closest ko access karta hai or is ky close main Function wala Variable hi hai.

Lexical Scope and Block Scope

Lexical Scope

Question: Lexical Scope kesy kehty hain?
Answer: Lexical Scope yeh hota hai ky Inner Function ka Scope. Phir us ky Outer Function ka Scope. Phir us ky Outer ka Scope isi tarah Global Scope tak. Yeh jitny b Scope hain in ko Lexical Scope kehty hain.

Note! Ager ap pury ka pura Function hi Return kar daty hoon tu sirf Function hi return nai hota, us ka pura Scope return hota hai. Or sirf Scope hi nai wo pura us ka Lexical Scope return hota hai.

Note! Outer function ky ander jitny b inner function hon un sab ko outer function ka access hota hai.
Example: 

        function outer(){
          let username = "Abdul Rehman";
          console.log(secret); // yeh variable (secret) inner function main define kia gya hai. is ko hum outer ky ander access 
                                  nai kar sakty. kio ky hum outer function ky ander inner function ko access nai kar sakty.
          function innerOne(){
              let secret = my123; // is variable ka access sirf isi inner function ky pass hai. Na hi outer function ky pass hai 
                                     or na hi kisi dosry inner function ky pass hai. jo ky hamary is code main dosray 
                                     inner functions (innerTwo and innerThree) hain.
              console.log("innerOne", username);
          }
          function innerTwo(){
              console.log("innerTwo", username);
              console.log(secret); // yeh b access nai ho ga. kio ky inner inner apas main b access nai hoty
          }

          }
          innerOne() // innerOne Abdul Rehman
          innerTwo() // innerTwo Abdul Rehman

      }

Block Scope

Question: Block Scope kesy kehty hain?
Answer: Block Scope yeh hota hai ky sirf Curly Bracket "{}" ka ander jo b code likha jye usy Block Scope Kehty hain.

Note! Block Scope ky ander jo b "Let" or "Const" sy "Variable" define Karyn gy wo isi Block Scope main access ho ga is ky bahir nai ho ga. But "Var" sy jo "Variable" define Karyn gy wo Block Scope ky bahir b access ho ga.

Closures Scope

Note! Ager hum Inner Function ky ander Outer Function ky Variables ko use karty hain tu wo Inner Function aik Closure Scope bana leta hai.

Note! Ager ap pury ka pura Function hi Return kar daty hoon tu sirf Function hi return nai hota, us ka pura Scope return hota hai. Or sirf Scope hi nai wo pura us ka Lexical Scope return hota hai.

Function Memory Creation

Note! Function ko jitni dafa Call karyn gy. Har dafa aik New Memory create ho gi.

Note! Jab Function ko Call karty hain tu wo apni aik Memory create karta hai. Or phir jab hum variables ko Console.log karyn gay tu Function delete ho jye ga. or Variables b delete ho jyen gy. Isi tarah jab hum Function ko again call karyn gy tu phir isi tarah same ho ga jesy yeh pechy bayan howa. Or har dafa new memory create ho gi.

Note! Jab Function run hota hai tu wo Variables create hoty hain jo Function main define kye hoon. Or jab Function End hota tu Variables b delete ho jaty hain.

Note! Jab koi Child Function us ky kisi Variables ky sath Closure bana leta hai tu wo immediately wahan sy delete nai hota. wo wahi par rehta hai. Or jo un ki location ka reference hai wo us Function (Child) ko dy dya jata hai. Or Function end ho gya lekin us ky Variables ko delete nai kia gya. wo Variables jo Child Function use kar raha hai.

Note! Ager hum ny Parent Function main 2 Variables define kye farz karyn (a aur b) name sy. Phir hum ny (a) ko Child Function main use kya or (b) ko Parent Function main use kya tu ho ga kuch is tarah ky (a) Variable ki memory create ho gi. Phir us Variable main jo Value define ki hai usy rakhy ga. Phir (b) Variable ki Memory create ho gi. Phir (b) ki value rakhy ga. Phir (b) Variable Console ho jye ga. Phir hum ny jo (a) Variable ko Child Function main use kia hai us ki memory wahan rahy gi. Phir jab hum Child Function ko return karyn gy tu (b) ki value delete ho jye gi lekin (a) ki value wahan rahy gi. wo delete nai ho gi. (a) ki value ka jo reference hai wo Child Function ko dy dya jye ga.

Note! jo b hum Variable ko Child Function main use karty hain tu Child function (inner function) aik Closure bana leta hai. Closure ka matlb yeh hai ky jab Function end ho jye ga to wo variables delete nai hoon gay wo memory main hi rahayn gay. Or un ka reference jo b un main function tha us ko dy dya jye ga. ager hum us function ko return b kar dety hain tu us ka reference wahan rahy ga wo delete nai ho ga.

Note! Ager hum Child Function ky around multiple Parents function create karyn or har Parent function main aik variable define karyn. phir hum ager Child Function main un sab variables ko use karyn tu un sab ka alag alag sy Closure scope bany ga
